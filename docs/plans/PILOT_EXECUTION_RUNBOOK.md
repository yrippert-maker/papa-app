# Pilot Execution Runbook — пошагово

**Цель:** провести демо 10–15 мин строго по flow. Не импровизировать.

---

## Перед демо (5 мин)

- [ ] Приложение запущено: **`npm run dev:stable`** → http://localhost:3001 (не `npm run dev`!)
- [ ] Иметь под рукой [DEMO_SCRIPT_V1.2.md](DEMO_SCRIPT_V1.2.md) — текст «что говорить» + checklist за 2 мин до созвона
- [ ] Вход: admin@local / пароль из AUTH_ADMIN_PASSWORD
- [ ] Документы проиндексированы: `npm run docs:index:agent` (если DATABASE_URL)
- [ ] Открыть дашборд → Помощник по документам

---

## Demo flow (строго по порядку)

### Шаг 1. Поиск (2 мин)

1. Ввести в поле поиска: **акт входного контроля ТВ3-117**
2. Нажать **Найти**
3. Показать результаты с **confidence %** (badge справа от названия)
4. Выбрать 2–3 документа (клик по карточке)
5. Убедиться, что intent = **Акт**

### Шаг 2. Черновик + auto-suggest (3 мин)

1. Нажать **Подготовить черновик (N)**
2. Показать автозаполненные поля (если есть — serial_number, inspector)
3. Показать label «(подсказка из документа)» у missing_fields
4. Дозаполнить оставшиеся поля вручную (если нужно)

### Шаг 3. EvidenceMap (2 мин)

1. Раскрыть источник (клик по строке)
2. Показать фрагмент + confidence %
3. Клик **Открыть документ** → проверить, что скачивается

### Шаг 4. Confirm → Export (2 мин)

1. Нажать **Подтвердить**
2. Нажать **Скачать DOCX**
3. Показать **sha256** под кнопкой

### Шаг 5. Аудит (опционально, 1 мин)

- Упомянуть: audit_meta, sources[].confidence, Evidence Kit для регулятора
