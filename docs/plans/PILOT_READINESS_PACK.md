# Pilot Readiness Pack — AI Document Workflow

**Версия:** 1.0 (baseline)  
**Дата:** 2026-02

---

## 1. Pilot scope (1 стр.)

| Параметр | Значение |
|----------|----------|
| **Изделие** | ТВ3-117 (двигатель) |
| **Тип документа** | Акт входного контроля |
| **Регулятор** | EASA Part-145 (или ARMAC — по выбору пилота) |
| **Окружение** | 1 MRO / CAMO / OEM-подрядчик |

**Цель пилота:** доказать сокращение времени подготовки акта и повышение прозрачности для аудитора.

---

## 2. Success criteria

| Критерий | Базовый уровень (до) | Целевой (после пилота) |
|----------|---------------------|-------------------------|
| **Время подготовки документа** | 30–60 мин (ручной поиск + заполнение) | &lt; 15 мин |
| **Количество ручных правок** | 100% полей вручную | 1–2 поля (остальное auto-suggest) |
| **Прозрачность источников** | «Откуда взято?» — неясно | EvidenceMap + confidence % — доказуемо |
| **Деградация при сбое ИИ/индекса** | Ошибка блокирует процесс | UI сообщает о деградации, доступен ручной режим |

**Дополнительно:** output_sha256, audit_meta.sources — воспроизводимость и целостность.

---

## 3. Demo flow (10–15 мин)

### Шаг 1. Поиск (2 мин)

1. Открыть Помощник по документам
2. Ввести запрос: *«акт входного контроля ТВ3-117»*
3. Получить результаты с **confidence %** (зелёный ≥80%, жёлтый ≥50%)
4. Выбрать 2–3 релевантных документа

### Шаг 2. Черновик + auto-suggest (3 мин)

1. Выбрать intent: **Акт**
2. Нажать «Подготовить черновик»
3. Проверить **автозаполненные поля** (serial_number, inspector — из документов)
4. Дозаполнить оставшиеся missing_fields (если есть)

*Если auto-suggest недоступен, пользователь продолжает работу в ручном режиме без потери данных.*

### Шаг 3. EvidenceMap (2 мин)

1. Раскрыть источник в EvidenceMap
2. Увидеть фрагмент + confidence %
3. Клик «Открыть документ» → скачать оригинал

### Шаг 4. Confirm → Export (2 мин)

1. Нажать «Подтвердить»
2. Скачать DOCX
3. Проверить **sha256** под кнопкой (целостность)

### Шаг 5. Аудит (опционально, 3 мин)

1. Показать audit_meta в regulatory bundle
2. sources[].confidence, embeddings provider, workflow_schema_version

## 4. Evidence Kit v1.2

На базе v1.1 + новые поля:

| Поле | Описание |
|------|----------|
| `sources[].confidence` | Релевантность источника (0–1) |
| `fieldSuggestions` | Какие поля заполнены из документов (опционально в audit) |
| `output_sha256` | Хэш финального DOCX, подтверждающий неизменность после экспорта |

**Сборка:** `npm run evidence-kit:public -- --tag evidence-kit-v1.2 --pdf`

---

## 5. Чего НЕ делать в пилоте

- ❌ Новые типы документов «на всякий случай»
- ❌ Новые AI-модели без запроса заказчика
- ❌ Мультимодальность / real-time / collaboration
- ❌ «Ещё один шаблон», если он не под пилот

**Фокус:** 1 изделие, 1 тип документа, 1 регулятор.

---

## 6. Контакты и следующие шаги

| Этап | Действие |
|------|----------|
| **До пилота** | Согласовать scope с заказчиком (EASA vs ARMAC) |
| **Пилот** | 1–2 недели, 3–5 реальных актов |
| **После** | Собрать метрики, решить: масштабирование или шаблон под регулятора |
