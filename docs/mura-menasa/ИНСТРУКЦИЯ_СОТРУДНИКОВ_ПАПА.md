# Инструкции для сотрудников — работа с ПАПА

**Приложение:** ПАПА (Поддержка Авиационных Производственных Активностей)  
**Версия:** 1.0  
**Дата:** 2026-02

---

## 1. Общие сведения

ПАПА — система управления документами, compliance и учётом для МРО Mura Menasa FZCO.

### Доступ

- **Web:** URL приложения (например, https://papa-app.example.com)
- **Логин:** по email и паролю (Credentials) или через провайдер (OAuth)
- **Роли:** определяют доступ к разделам и функциям

---

## 2. Инструкции по ролям

### 2.1 ADMIN (Администратор)

**Доступ:** полный. Управление пользователями, настройками, системой.

**Основные действия:**
- **Пользователи** (`/admin/users`) — добавление, редактирование, роли, сброс паролей
- **Система** (`/system/settings`) — конфигурация, здоровья сервисов
- **Audit** — просмотр событий, экспорт

**Рекомендации:** не выполнять операционные действия за других ролей; хранить логи и backup.

---

### 2.2 MANAGER (Менеджер)

**Доступ:** операции, документы, ledger, TMC, compliance.

**Основные действия:**
- **Дашборд** (`/`) — обзор, Помощник по документам (AI)
- **Документы** — Библиотека, Финансы, Mura Menasa, Техкарты
- **Compliance** (`/compliance/decisions`) — одобрения, ключи, inbox
- **TMC** (`/tmc/lots`) — склад, заявки
- **Ledger** — просмотр и добавление событий

**Помощник по документам:**
1. Поиск документов — ввести запрос (например, «ТВ3-117»)
2. Выбрать тип: Акт, Техкарта, Бланк MM, Письмо
3. Для актов — можно загрузить фото, AI распознает поля
4. Подготовить черновик → подтвердить → скачать DOCX

---

### 2.3 ENGINEER (Инженер)

**Доступ:** просмотр документов, TMC (чтение), заявки TMC.

**Основные действия:**
- **Документы** — Библиотека регуляторики, Mura Menasa
- **Техкарты** — эталоны, ссылки на MOPM
- **TMC** — просмотр заявок, lots
- **Помощник** — поиск и формирование техкарт, актов (при FILES.LIST)

**Ограничения:** нет загрузки файлов, нет ledger append, нет compliance manage.

---

### 2.4 STOREKEEPER (Кладовщик)

**Доступ:** TMC, файлы, ledger.

**Основные действия:**
- **TMC** (`/tmc/lots`, `/tmc-requests/incoming`) — создание lots, обработка заявок
- **AI Inbox** — загрузка документов для обработки
- **Ledger** — append событий

**Рекомендации:** сверять остатки, вести учёт по партиям.

---

### 2.5 AUDITOR (Аудитор)

**Доступ:** только чтение. Compliance, audit, ledger, документы.

**Основные действия:**
- **Audit** (`/audit`) — события, экспорт
- **Compliance** — просмотр решений, политик
- **Документы** — просмотр
- **Ledger** — чтение

**Ограничения:** нет изменений, нет управляющих действий.

## 3. Типовые сценарии

### 3.1 Сформировать акт АВК по фото

1. Дашборд → Помощник по документам
2. Выбрать «Акт»
3. Нажать «Фото акта» → загрузить PNG/JPG
4. (Требуется OPENAI_API_KEY для распознавания)
5. Подготовить черновик → заполнить недостающие поля → Подтвердить → Скачать DOCX

### 3.2 Найти документ и сформировать техкарту

1. Документы → Техкарты
2. Выбрать нужный эталон (ТВ3-117, АИ-9, НР-3)
3. Помощник → поиск «ТВ3-117» → выбрать «Техкарта»
4. Выбрать документы из результатов → Подготовить черновик
5. Заполнить операцию, шаги, критерии приемки → Экспорт

### 3.3 Реестр платежей (Менеджер)

1. Документы → Финансы
2. Реестр платежей — сводная таблица, ежеквартальная сводка для аудитора
3. Данные поступают из очереди почты после одобрения

### 3.4 Compliance — одобрение

1. Compliance → Inbox / Decisions
2. Просмотр предложений
3. Accept / Reject с указанием причины

- Одобрения, задачи, критические события
- См. `docs/ops/TELEGRAM_BOT.md`

## 5. ЭЦП с биометрией

При включённой настройке (`BIOMETRIC_SIGN_ENABLED=1` или `config/ecp-biometric.json`) доступно подписание документов с подтверждением отпечатком (WebAuthn). Роли: ADMIN, MANAGER, ENGINEER.

**API:** `POST /api/signature/biometric` — body: `{ documentHash, assertionVerified: true }`.  
См. `docs/ops/ECP_BIOMETRIC.md`.
