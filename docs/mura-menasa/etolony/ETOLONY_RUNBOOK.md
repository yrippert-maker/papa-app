# Runbook: формирование эталонов через агент

## Шаг 1 — Индексация документов-источников

**Образцы документов:**
- `Новая папка/` — общие документы (PAPA, регламенты, RK_MMF и т.д.)
- `Новая папка/menasa/` — Mura Menasa: Руководство по ТОиР, СУБП, РЦ-АП-145.11, ТУ_ММ, TV3-117_Tech_Cards, ai9 и др.

В терминале (из корня проекта):

```bash
DOCS_ROOT_DIR="/Users/yrippertgmail.com/Desktop/papa-app/Новая папка" npm run docs:index:pgvector
```

Индексация рекурсивная — `menasa/` включается автоматически.

Если PDF тяжёлые:

```bash
DOCS_AGENT_SKIP_PDF=1 DOCS_ROOT_DIR="/Users/yrippertgmail.com/Desktop/papa-app/Новая папка" npm run docs:index:pgvector
```

## Шаг 2 — В UI: «Помощник по документам» (Dashboard)

### Команда 1 (поиск источников)

В поле поиска введите:

> «Найди: технологии ремонта ТВ3-117, руководство по капитальному ремонту, требования СУБП/качества Mura Menasa, упоминания ИКАО/EASA/ARMAC»

Выберите найденные документы чекбоксами.

### Команда 2 (черновик эталона)

1. Выберите тип: **Акт (входной/выходной контроль)**
2. Нажмите **«Подготовить черновик»**
3. Заполните поля (S/N, inspector, approver и т.д.)
4. Нажмите **«Подтвердить»**
5. Нажмите **«Скачать DOCX»**

### Цикл по типам эталонов

| Поиск | Тип | Сохранить как |
|-------|-----|---------------|
| акт входного контроля ТВ3-117 | Акт | `Акт_входного_контроля_TV3-117_Эталон.docx` |
| акт выходного контроля ТВ3-117 | Акт | `Акт_выходного_контроля_TV3-117_Эталон.docx` |
| акт выбраковки | Акт | `Акт_выбраковки_TV3-117_Эталон.docx` |
| претензионное письмо | Письмо | `Претензионное_письмо_TV3-117_Эталон.docx` |
| … | … | … |

## Шаг 3 — Размещение

Скопируйте скачанные DOCX в соответствующие папки:

- `docs/mura-menasa/etolony/TV3-117/03_Акты_входного_контроля/`
- `docs/mura-menasa/etolony/TV3-117/04_Акты_выходного_контроля/`
- и т.д.

## Шаг 4 — Внутреннее утверждение

Перед использованием эталонов в production:

1. Проверка соответствия ИКАО/EASA/ARMAC
2. Утверждение quality/compliance
3. Обновление матрицы соответствия (`01_Матрица_соответствия_требованиям.md`)
