# Регламент обновления инструкций (NFR-9.1)

**Версия:** 1.0  
**Дата:** 2026-02-03  
**Статус:** Утверждён

## 1. Область применения

Данный регламент распространяется на:
- Инструкции пользователя ПАПА (руководства, FAQ, видеоинструкции)
- Документацию операционных процедур (runbook)
- Внутренние регламенты Mura Menasa FZCO

## 2. Ответственность

| Роль | Обязанности |
|------|-------------|
| Ответственный за документацию | Планирование обновлений, сбор замечаний |
| Руководитель подразделения | Согласование изменений |
| Администратор ПАПА | Публикация обновлений в системе |

## 3. Инициация обновления

3.1. Основания для обновления:
- Изменение функциональности ПАПА
- Выявленные ошибки или неточности в инструкциях
- Изменение нормативных требований (АП-145, EASA, ICAO)
- Плановая проверка (раз в квартал)

3.2. Заявка оформляется в виде issue или служебной записки с указанием:
- Раздел/документ для обновления
- Причина изменения
- Предлагаемый текст или описание правок

## 4. Процедура обновления

4.1. **Подготовка**
- Ответственный подготавливает черновик изменений
- Черновик согласовывается с заинтересованными лицами

4.2. **Версионирование**
- Каждое обновление получает версию (MAJOR.MINOR)
- MAJOR — существенные изменения структуры или содержания
- MINOR — правки, дополнения, уточнения

4.3. **Публикация**
- Обновлённый документ размещается в репозитории/docs
- При необходимости обновляется индекс (config/video-instructions.json, DOCUMENT_INDEX)
- Дата обновления фиксируется в заголовке документа

## 5. Периодичность проверки

| Тип документа | Периодичность | Ответственный |
|---------------|---------------|---------------|
| Инструкции пользователя | ежеквартально | Документатор |
| Runbook | при изменении процедур | Администратор |
| Видеоинструкции | при изменении UI | Документатор |

## 6. Связь с другими документами

- `docs/ops/LTS_POLICY.md` — политика поддержки версий
- `config/video-instructions.json` — каталог видеоинструкций
- `docs/QUICKSTART_CURSOR.md` — быстрый старт для разработчиков
