# NFR-8.2/8.3: Юридическая проверка ЭЦП с биометрией

**Статус:** Вне кода — требует юридического заключения и сертификации  
**Дата:** 2026-02  
**Связанные требования:** NFR-8.2 (сертификация биометрии), NFR-8.3 (соответствие законодательству)

---

## 1. Текущая реализация в коде

| Компонент | Реализация | Путь |
|-----------|------------|------|
| WebAuthn | Да | lib/webauthn, /settings/biometric |
| Фиксация подписанта | Да | signer_name, signer_role, signed_at в ledger |
| Журнал подписаний | Да | LEDGER_APPEND, appendLedgerEvent |
| Проверка подлинности | Да | Challenge → assertion → верификация на сервере |

**Используемые технологии:** Web Authentication API (FIDO2/WebAuthn), Touch ID / Windows Hello / Android биометрия.

## 2. Чеклист для юридической проверки (NFR-8.2)

### 2.1. Сертификация биометрии

- [ ] Определить применимое законодательство (РФ: 63-ФЗ, УКЭП; ОАЭ: местные нормы; EASA/ICAO при авиационной сфере)
- [ ] Проверить соответствие WebAuthn требованиям к биометрической аутентификации
- [ ] Получить заключение: признаётся ли WebAuthn-подпись аналогом УКЭП/квалифицированной ЭП в юрисдикции Mura Menasa FZCO
- [ ] При необходимости — сертификация у аккредитованного УЦ или органом по сертификации

### 2.2. Документы для юриста

- [ ] Описание архитектуры подписи: `docs/ops/EVIDENCE_SIGNING.md`, `lib/evidence-signing.ts`
- [ ] Схема потока: challenge → assertion → сохранение в ledger
- [ ] Перечень фиксируемых данных: ФИО, роль, дата/время, идентификатор credential

---

## 3. Чеклист для соответствия законодательству (NFR-8.3)

### 3.1. РФ (63-ФЗ, УКЭП)

- [ ] Является ли подпись через WebAuthn «аналогом собственноручной подписи» по ст. 160 ГК РФ
- [ ] Требуется ли использование УКЭП (квалифицированная ЭП) для документов, регулируемых 63-ФЗ
- [ ] Совместимость с требованиями к хранению доказательств подписания (срок, неизменяемость)

### 3.2. ОАЭ / Mura Menasa FZCO

- [ ] Применимое законодательство ОАЭ для электронной подписи
- [ ] Требования к признанию биометрической подписи в судах и органах
- [ ] GDPR/локальные нормы защиты персональных данных (биометрия)

### 3.3. Авиационная отрасль (EASA, ICAO, АП-145)

- [ ] Требования к подписи производственных документов (акты, техкарты)
- [ ] Допустимость WebAuthn для документов, подпадающих под Part-145 / АП-145

## 4. Рекомендуемые действия

1. **Назначить ответственного** — юрист или compliance-офицер Mura Menasa FZCO
2. **Собрать пакет документов** — архитектура, flow-диаграммы, примеры сохранённых событий ledger
3. **Заказать юридическое заключение** — у профильной юридической фирмы (IT/авиация)
4. **Зафиксировать результат** — дата заключения, вывод (признано/не признано/с оговорками)
5. **Обновить СООТВЕТСТВИЕ_ТЗ** — после получения заключения пометить NFR-8.2/8.3 выполненными
