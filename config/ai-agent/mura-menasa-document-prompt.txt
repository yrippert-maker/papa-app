# Контекст для генерации документов Mura Menasa FZCO

При создании документов для Mura Menasa соблюдать:

## Few-shot примеры (FR-2.3)

### Акт входного контроля — заполненные поля
```
act_type: входного контроля
product: ТВ3-117
serial_number: 12345-678
act_number: АВК-2025/001
act_date: 2025-02-03
inspector: Иванов И.И.
approver: Петров П.П.
decision: Годен
```

### Техкарта — шаблон полей
```
operation: Разборка двигателя
product: ТВ3-117
version: 1.0
steps: 1. Отключить заглушки. 2. Снять крепёж. 3. ...
acceptance_criteria: Комплектность сохранена
requirement_ref: REQ:MM-QM-§4.2; AП-145
```

### Бланк фирмы — письмо
```
org_name: MURA MENASA FZCO
recipient: ООО Контрагент
subject: По вопросу поставки
```

## Стиль (брендбук)
- Основной цвет: #EF1C23 (красный Mura Menasa)
- Логотип: использовать с /mura-menasa-logo.png
- Шрифты: Inter, Arial, Times New Roman
- Фирменный бланк: templates/mura-menasa-firm-blank.docx

## Регуляторные требования
- Руководство по качеству Mura Menasa (РК)
- Руководство по процедурам ТОиР (MOPM) Mura Menasa
- АП-145 авиарегистра МАК
- EASA
- ICAO

## Правила
- Все служебные документы — на фирменном бланке
- Акт входного/выходного контроля — по формату MOPM
- Технологические карты — по ТР ММ-00002-00001
- Не выдумывать данные — запрашивать недостающие поля

Полный гайд: docs/mura-menasa/BRANDBOOK_AI_GUIDE.md

## Эталонные шаблоны документов

Все документы генерируются на основе эталонных шаблонов из templates/docx/brandbook/.
Маппинг типов документов:
- Тип «Бланк MM» / «Письмо» → 01_Фирменный_бланк_MM.docx
- Тип «Акт» (входной контроль, АВК) → 02_АВК_шаблон_MM.docx
- Тип «Акт» (выходной контроль, АВыхК) → 03_АВыхК_шаблон_MM.docx
- Тип «Техкарта» → 04_Техкарта_шаблон_MM.docx

При генерации документа:
1. Определи тип документа по запросу пользователя
2. Загрузи соответствующий шаблон
3. Заполни поля данными из draftFields
4. Сохрани в формате брендбука (цвет #EF1C23, шрифт Arial, логотип в шапке)
